## 最小完整 Table 結構

```dbml
Table <PascalCaseTableName> {
  <pk_column>   <sql_type>    [pk, label: '<UI標籤>', group: '<分組>']
  <col_name>    <sql_type>    [not null, label: '<UI標籤>']
  <sensitive_col> <sql_type>  [not null, sensitive: true, label: '<UI標籤>']
  <status_col>  nchar(1)      [not null, default: '<val>', ref_code: '<CodeKey>', label: '<UI標籤>']
  <fk_col>      <sql_type>    [label: '<UI標籤>']

  Note: '<系統代碼> <業務說明>'
}

Ref: <TableA>.<fk_col> > <TableB>.<pk_col>
```

## v3.3 四個一級標註快速參考

```dbml
// label: 業務顯示名稱（UI 用）
userId nvarchar(60) [pk, label: '使用者帳號']

// ref_code: 動態列舉來源 key（CodeMain 查詢）
status nchar(1) [ref_code: 'UserStatus', label: '狀態']

// sensitive: true（密碼/電話/email/身分證等敏感欄位）
password nvarchar(150) [sensitive: true, label: '密碼']

// group: 邏輯分組（haPDL 可以據此分頁籤/分欄位組）
deptId nvarchar(20) [group: 'org', label: '部門']
```

## SQL 型別慣例

| 用途 | SQL 型別 |
|------|---------|
| 業務 ID / 代碼（短）| `nvarchar(20)` |
| 使用者帳號 / UUID | `nvarchar(60)` |
| 名稱 / 標題 | `nvarchar(100)` |
| 描述 / 備註（長）| `nvarchar(500)` |
| 狀態碼（1碼）| `nchar(1)` |
| 密碼 hash | `nvarchar(150)` |
| 金額 | `decimal(18,2)` |
| 整數 | `int` |
| 時間戳記 | `datetime2(0)` |
| 布林 | `bit` 或 `nchar(1)` |

## 反模式速查（AP-01~AP-05）

| 反模式 | 說明 |
|--------|------|
| AP-01 | 用 `note: 'label:...|group:...'` 嵌入，改用一級語法 |
| AP-02 | 同欄位 `ref_code:` 與 `ref:` 並存，二擇一 |
| AP-03 | 顯式寫 `sensitive: false`，省略即可 |
| AP-04 | DBML 和 haPDL 重複定義同名 group，只在 DBML 定義一次 |
| AP-05 | color/icon/display_type 放在 DBML 中，移到 haPDL |
