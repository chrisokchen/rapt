# User Journeys

> 來源：01-stakeholders.md、訪談記錄 1-3、PRD §1-§8
> 範圍：僅業務流程描述，不涉及技術實作

## consumer 的主要旅程 1：發現與購買

1. **進入商店並尋找商品**
   - 觸發：行銷活動、口碑、回購需求
   - 動作：瀏覽分類（多層結構）、搜尋商品、查看商品頁（含原價/特價與庫存狀態）
   - 預期結果：快速（2 秒內）找到想買的商品，且庫存狀態真實可信

2. **加入購物車與湊單**
   - 觸發：決定購買或先收藏
   - 動作：加入購物車、調整數量；為達免運門檻湊單（滿 1,000 免運的誘因行為）
   - 預期結果：購物車跨裝置保留

3. **結帳**
   - 觸發：購物車內容確定
   - 動作：確認商品 → 填寫/選擇收件資訊（可用常用地址）→ 選擇付款方式（信用卡/PayPal/轉帳）→ 套用優惠（券、點數、會員折扣）→ 確認金額（含運費、5% 稅）→ 提交訂單
   - 預期結果：必填欄位最少；折扣規則透明、金額自動算對

4. **付款**
   - 觸發：訂單已提交
   - 動作：透過金流完成付款；轉帳者匯款後回報帳號後五碼
   - 預期結果：付款安全（支付資訊不落地）；付款成功即確定保留到貨

5. **追蹤與收貨**
   - 觸發：訂單狀態變更
   - 動作：收到 LINE/Email 通知（已確認、已出貨、配送中）；隨時可查物流進度
   - 預期結果：全程進度可見，不需打電話詢問

6. **累點與回購**
   - 觸發：訂單完成
   - 動作：獲得點數（消費 100 元得 1 點）、會員等級累計
   - 預期結果：下次消費可折抵（1 點折 1 元）、高等級享更多優惠

## consumer 的主要旅程 2：退換貨

1. **申請退貨**
   - 觸發：商品不符期待，且在退貨期限內
   - 動作：在手機上對已完成訂單按「申請退貨」，免紙本表單
   - 預期結果：申請流程幾分鐘內完成

2. **等待審核與取件**
   - 觸發：商家後台審核通過
   - 動作：物流上門收件
   - 預期結果：不必自行寄回

3. **收到退款**
   - 觸發：商品退回商家確認
   - 動作：（無，等待）
   - 預期結果：退款到帳並收到通知

## merchant 的主要旅程 1：開店與商品上架

1. **註冊開店**
   - 觸發：決定採用月租方案
   - 動作：註冊商家帳號、設定店面基本資料
   - 預期結果：當天就能開始上架（開箱即用）

2. **建立商品**
   - 觸發：店內商品要搬上線
   - 動作：建立商品（名稱/描述/分類/品牌/圖片）、設定規格組合（顏色×容量，每組合一個 SKU 與售價）、設定原價（划線價）、存草稿或上架
   - 預期結果：不需技術知識即可完成

3. **設定庫存**
   - 觸發：商品上架
   - 動作：以 SKU 為單位輸入庫存、設定低水位補貨提醒
   - 預期結果：之後賣一個自動扣一個，不再人工對 Excel

## merchant 的主要旅程 2：日常接單出貨

1. **檢視訂單看板**
   - 觸發：每日營運
   - 動作：一個畫面依狀態（待付款/已付款/該出貨）篩選訂單
   - 預期結果：不漏單、不重複出貨

2. **確認與出貨**
   - 觸發：訂單已付款
   - 動作：查看明細、揀貨、交給貨運行、標記出貨並填物流單號
   - 預期結果：消費者自動收到出貨通知，不再被電話追問

3. **處理退貨申請**
   - 觸發：消費者提出退貨
   - 動作：後台審核（核准/拒絕）、核准後系統安排取件、確認退回商品後執行退款
   - 預期結果：有明確規則可循（期限、排除品項），不被濫退

4. **查看報表**
   - 觸發：進貨決策、月結
   - 動作：查看銷售概況（訂單數/營業額/客單價）與商品排行（暢銷/滯銷）
   - 預期結果：「看得懂就好」的極簡報表，作為進貨依據

## merchant / platform-marketer 的旅程：促銷活動操作

1. **規劃活動**
   - 觸發：節慶檔期、滯銷品出清；商家自操或行銷人員代操
   - 動作：選擇活動類型（百分比折扣/滿額折現/買 X 送 Y）、設定檔期起迄
   - 預期結果：時間到自動上下架，不靠人工切

2. **發行優惠券**
   - 觸發：拉新或回購行銷
   - 動作：建立優惠券（代碼、折扣方式、最低消費門檻、總張數上限、有效期間、限定會員等級）
   - 預期結果：防濫用機制齊備，不被外流薅羊毛

3. **檢視成效**
   - 觸發：活動結束
   - 動作：查看活動期間銷售變化
   - 預期結果：能判斷活動是否有效（轉換率拉高三成的承諾）

## platform-admin 的旅程：商家管理（輪廓粗略）

1. **開通商家** — 商家註冊後審核/開通月租帳號
2. **監控營運** — 追蹤商家活躍度、留存（構想書 KPI）
3. **處理異常** — 停權違規商家
