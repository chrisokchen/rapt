## 中小零售店老闆 / 商家的主要旅程：建立並營運線上商店

1. **建立商品與銷售單位**
   - 觸發：商家要把實體店商品搬到線上販售。
   - 動作：商家建立商品名稱、描述、分類、品牌、圖片與不同規格組合。
   - 預期結果：每個可銷售規格都有唯一 SKU 與獨立售價，商品可草稿、上架或下架。
   - 對應系統功能：商品管理、SKU 管理、分類管理。

2. **設定庫存與補貨水位**
   - 觸發：商品準備販售或庫存異動。
   - 動作：商家以 SKU 為單位維護庫存，設定低庫存通知水位。
   - 預期結果：商品頁呈現正確庫存狀態，庫存不足時提醒商家。
   - 對應系統功能：庫存管理、低庫存通知。

3. **設定促銷、優惠券與會員權益**
   - 觸發：商家或平台行銷要推動銷售。
   - 動作：設定全館或分類折扣、滿額折現、買 X 送 Y、優惠券門檻、張數上限與會員等級限制。
   - 預期結果：活動在指定檔期自動生效與失效，折扣規則可被正確計算。
   - 對應系統功能：促銷活動、優惠券、會員分級。

4. **處理訂單與出貨**
   - 觸發：消費者完成下單或付款。
   - 動作：商家查看訂單狀態，依狀態出貨並填入物流單號。
   - 預期結果：商家知道哪些訂單待付款、已付款、該出貨或已完成；消費者能追蹤物流。
   - 對應系統功能：訂單管理、物流追蹤、狀態通知。

5. **查看營運報表**
   - 觸發：商家要判斷銷售表現與補貨策略。
   - 動作：查看期間訂單數、營業額、客單價、暢銷與滯銷商品。
   - 預期結果：商家可用簡單報表做進貨與活動判斷。
   - 對應系統功能：基本報表。

## 一般網購消費者的主要旅程：從瀏覽到售後

1. **尋找商品**
   - 觸發：消費者有購物需求或看到促銷。
   - 動作：用手機瀏覽商店、搜尋商品、查看分類與商品資訊。
   - 預期結果：頁面快速載入，商品容易找到，價格與折扣資訊清楚。
   - 對應系統功能：商品瀏覽、搜尋、分類、特價顯示。

2. **收藏或加入購物車**
   - 觸發：消費者對商品有興趣但不一定立即購買。
   - 動作：加入願望清單或購物車，調整數量或移除商品。
   - 預期結果：購物車跨裝置保留，願望清單可作為日後購買參考。
   - 對應系統功能：願望清單、購物車。

3. **結帳與付款**
   - 觸發：消費者決定購買。
   - 動作：確認商品、填寫或選擇收件資訊、選擇付款方式、套用優惠並提交訂單。
   - 預期結果：必填欄位少，付款方式符合需求，金額、運費、稅金與折扣清楚。
   - 對應系統功能：結帳、付款、優惠套用、稅金與運費計算。

4. **追蹤訂單與配送**
   - 觸發：訂單成立、付款完成或商家出貨。
   - 動作：查看訂單狀態，接收 LINE 或 Email 通知。
   - 預期結果：消費者知道訂單是否確認、出貨、配送中或送達。
   - 對應系統功能：訂單查詢、物流追蹤、狀態通知。

5. **申請退貨**
   - 觸發：消費者收到商品後需要退貨。
   - 動作：在線上對已完成訂單申請退貨，等待商家審核與物流取件。
   - 預期結果：不需紙本表單，退貨核准後可安排取件並完成退款。
   - 對應系統功能：退貨申請、退貨審核、退款流程。

## 平台營運 / 行銷人員的主要旅程：協助商家做促銷

1. **規劃促銷工具**
   - 觸發：平台要提供商家常用的電商活動能力。
   - 動作：定義百分比折扣、固定金額折抵、買 X 送 Y 與優惠券限制。
   - 預期結果：商家能用標準工具建立活動，避免人工操作。
   - 對應系統功能：促銷活動與優惠券。

2. **設計會員分級行銷**
   - 觸發：平台希望支援 VIP 專屬權益與高等級會員券。
   - 動作：規劃會員等級、優惠資格與專屬活動。
   - 預期結果：不同會員等級可取得不同權益或優惠券資格。
   - 對應系統功能：會員分級、優惠券資格。

3. **控管折扣風險**
   - 觸發：多種折扣同時存在可能造成商家損失。
   - 動作：定義可疊加、互斥與計算順序。
   - 預期結果：結帳金額可預測，商家不會因促銷規則不清而虧損。
   - 對應系統功能：折扣計算規則。
